顶级语句概述

在上一篇笔记中，我们介绍了 C# 9 引入的顶级语句。

只由顶级语句构成的程序并不是“没有 Main”，当我们不写 Main的情况下，编译器帮你生成一个 Main。

由于顶级语句是 C# 9 的语言特性，不能被关闭，所以我们在实际编写代码的过程中，要注意以下问题，避免报错。

顶级语句 + Main

在C#程序的运行中，顶级语句 + Main不一定会冲突报错
实际规则如下：
如果存在顶级语句：

  • 顶级语句生成的 Main 是唯一入口
  • 您手写的 Main 不会作为入口
  • 但它可以合法存在，不报错

入口错误

如果在一个项目中，同时出现多个“顶级语句文件”

同一项目中，我们写了两个顶级语句文件，A.cs 和 B.cs。
当我们运行这个项目时，会报错，这是因为编译器无法确定哪个文件是入口文件，所以报错。

推荐结构

纯顶级语句

在实际项目中，我们不写 Main 方法，所有逻辑顺序执行，也不在别处引用顶级变量。

传统 Main

这种写法中，我们不写任何顶级语句,所有逻辑在 Main 或它调用的方法中。
