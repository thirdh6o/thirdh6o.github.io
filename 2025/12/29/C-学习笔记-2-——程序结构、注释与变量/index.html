掌握C#程序结构

下面是一个简单的C#程序结构示例：

using System 使用系统命名 System 空间中的类。
namespace 名称空间是用来组织代码的容器，它是类和其他名称空间的容器。
class 类是数据和方法的容器，它为程序带来了功能。在C#中运行的每一行代码都必须在一个类中。在我们的示例中，我们将类命名为Program。

如果您学习过Java 或其他面向对象的编程语言，您会注意到C#的类与Java的类有一些相似之处。

Console 是System系统名称空间的一个类，它有一个WriteLine() 方法，用于输出/打印文本。

顶级语句

顶级语句（Top-level statements）可以把“控制台程序的样板代码”省掉。

在这个示例中，我们只写了一行代码，它会自动输出”Hello, World!”。
这个特性从 C# 9 开始支持，.NET 6/7/8 的控制台模板默认就用它。

隐式引入

如果您观察细心的话，您会注意到在上面的程序中，using System 这一行是灰色的，看起来并没有被使用。
理论上，我们的代码调用了System.Console.WriteLine() 方法，应该需要 using System。
这是因为现代 .NET 项目默认启用了「隐式 using」。从 .NET 6 / C# 10 开始，编译器会自动帮你加上一些常用的 using 语句，例如：

所以即使我们自己没有写 using System，编译器也会自动加上它，这就是它变成灰色的原因，编译器并没有使用这行语句。

WriteLine 或 Write

这是我们目前使用的两个输出方法，它们的区别在于是否换行。

您可以根据需要选择使用这两个方法。

注释

注释是用来解释代码的，它不会被编译器执行。
在C#中，有两种类型的注释：

单行注释

单行注释以 // 开头，直到行尾结束。

多行注释

多行注释以 /* 开头，以 */ 结尾。
